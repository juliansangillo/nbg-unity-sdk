Description

Keeps track of the active scene and abstracts the process for navigating between scenes using Unity's built in SceneManagement API. It will use the list of scenes added in Unity's Build Settings as the levels to track. If you want a scene to be treated as a level that can be accessed by the Level Manager, add it to the build settings! The filenames of the scenes should be unique, even if they are located in separate folders! To use, inject an ILevelManager object wherever needed using Zenject's "Inject" attribute.

A Level Manager not only keeps track of the levels, it also initializes a list of labels (display names) from the actual names of the levels. This allows you to reuse the level's name for UI display purposes. For example, "UltimateLevel8-1_005" will be mapped to the label "Ultimate Level 8-1 005". Below are the label rules.

  • Any set of letters starting with a capital is delimited with a space, except for the first one
  • Any set of numbers and/or special characters before the next capital or underscore is treated as its own word and delimited with a space
  • All underscores are replaced with spaces

◆ LoadFirstLevel()

void NaughtyBiker.LevelManagement.Interfaces.ILevelManager.LoadFirstLevel ( )

Loads the first level, which is index 0 by default, but can be changed by setting the FirstLevel property. This is useful if you want to bring a player back to the first level repeatedly during the game, but you don't want to repeatedly give the level name. It is also useful to be able to choose which level is the "first level" because index 0 is typically the main menu in a lot of games, which may not be where you actually want to bring the player.

Implemented in NaughtyBiker.LevelManagement.LevelManager.

void NaughtyBiker.LevelManagement.Interfaces.ILevelManager.LoadLevel ( string  name)

Loads the level (or scene) selected by name and sets that as the active level, while unloading the current active level. This allows you to jump to whichever level you need at any point during gameplay.

◆ LoadNextLevel()

void NaughtyBiker.LevelManagement.Interfaces.ILevelManager.LoadNextLevel ( )

Loads next level according to the scene's build index and the order the scenes are listed in Build Settings. This allows you to let LevelManager determine the next level and take you there automatically, in the case that levels are progressed sequentially in your game, without having to keep track of all level names. If there is no next level, this will take you back to the first scene listed in Build Settings.

Implemented in NaughtyBiker.LevelManagement.LevelManager.

◆ LoadPreviousLevel()

void NaughtyBiker.LevelManagement.Interfaces.ILevelManager.LoadPreviousLevel ( )

Loads previous level according to the scene's build index and the order the scenes are listed in Build Settings. This allows you to let LevelManager determine the previous level and take you there automatically, in the case that levels are progressed sequentially in your game, without having to keep track of all the level names. If there is no previous level, this will raise an InvalidOperationException.

Exceptions
InvalidOperationExceptionRaised when you are already on level index 0 and there is no previous level to load
